===== ath10k mesh mode =====

Refer this link for IEEE 802.11s (Mesh Networking) introduction.
[[https://wireless.wiki.kernel.org/en/developers/Documentation/ieee80211/802.11s]] 

ath10k supports mesh BSS (MBSS) in raw-mode since Sep/2015 (commit id: b6c7bafa7d4b1398cce93e4af0a48603919fa933) and in native Wi-Fi mode since Nov/2015 (commit id: bb58b89c5ea1006c93270fd2e667ab72312bf7ab).

Firmware must have “raw-mode” feature flag to use MBSS in raw-mode and “mfp-support” feature flag to use MBSS with security.


===== Bring up Mesh Point (MP) =====

There are 2 manners to bring up MP which are using “iw” and “wpa_supplicant”.

iw does MLME in kernelspace while wpa_supplicant does it in userspace.
iw is simple and quick to verify MP working because it doesn't require any other utilities, but cannot be used for secured mode due to lack of SAE support.

Contrarily wpa_supplicant may be a bit complex comparing to iw, however it is good to use with security.

===== Setup open Mesh Point using iw =====

In this example we will configure a MP to use an open Mesh network.
The MP will automatically peer with others that are using same configuration (mesh ID and channel).

Create a new MP interface "mesh0".
<code>iw phy phy0 interface add mesh0 type mp</code>

Confirm that the new interface created as MP.
<code>iw mesh0 info
Interface mesh0
        type mesh point</code>

Configure channel information.
<code>iw dev mesh0 set freq 5745 80 5775</code>

If you need, you can change interface’s MAC address.
<code>ifconfig mesh0 hw ether $MAC_ADDRESS</code>

Assign an IP and bring up MP.
<code>ifconfig mesh0 192.168.1.10</code>

Join a Mesh network.
“mesh-ath10k” is Mesh ID which can be maximum of 32 bytes long.
<code>iw dev mesh0 mesh join mesh-ath10k</code>

Verify peer link status
<code>iw mesh0 station dump

===== Setup Mesh Point using wpa_suppliant =====

Both of open MP and secured MP can be brought up using wpa_supplicant.
This section will instruct the steps to both of open and secured MP setup.

Create a new MP interface "mesh0".
<code>iw phy phy0 interface add mesh0 type mp</code>

Assign an IP and bring up MP.
<code>ifconfig mesh0 192.168.1.10</code>

Create a wpa_supplicant.conf file and configure Mesh information.

    * ssid: mesh ID
    * mode=5: mesh mode
    * key_mgmt=NONE: open Mesh
    * key_mgmt=SAE: secured Mesh
    * user_mpm=1: MPM on userspace

<code>ctrl_interface=/var/run/wpa_supplicant
user_mpm=1
network={
	ssid="mesh-ath10k"
	mode=5
	frequency= 5745
	key_mgmt=NONE 
}</code>

To configure secured MP, set key_mgmt to SAE and add psk value
<code>…
key_mgmt=SAE
psk=”123456789a”
…</code>

The wpa_supplicant requires the ieee80211w to be set to non-zero value to have secured mesh operating properly if both patches "mac80211: Encrypt "Group addressed privacy" action frames" and "ath10k: fix group privacy action frame decryption for qca4019" are not available.

<code>…
ieee80211w=1
…</code>

Run wpa_supplicant
<code>wpa_supplicant -D nl80211 -i mesh0 -c wpa_supplicant.conf –B</code>

Verify peer link status
